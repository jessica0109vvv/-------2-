# 生活部落格專案規格說明

## 專案目標
建立一個純前端的生活部落格網站，可以輕鬆記錄生活點滴，並且能夠直接部署到 GitHub Pages 上。使用者可以發表文章、上傳圖片、留言互動，並且支援文章的按讚和收藏功能。

## 技術棧
- HTML5
- CSS3
- JavaScript (原生)
- Bootstrap 5.3.0 (CDN 版本)
- Font Awesome 6.0.0 (CDN 版本)
- localStorage 用於資料儲存

## 資料結構
### 1. 文章資料 (blog-posts)
```javascript
{
  id: number,          // 文章唯一識別碼（時間戳）
  title: string,       // 文章標題
  content: string,     // 文章內容
  image: string,       // 圖片（base64 格式）
  date: string,        // 發表日期（ISO 格式）
  comments: array,     // 留言陣列
  likes: number,       // 按讚數
  liked: boolean,      // 當前用戶是否按讚
  favorited: boolean,  // 當前用戶是否收藏
  userId: number,      // 作者ID
  userEmail: string    // 作者郵箱
}
```

### 2. 使用者資料 (blog-users)
```javascript
{
  id: number,          // 用戶唯一識別碼
  email: string,       // 電子郵件
  password: string,    // 加密後的密碼
  createdAt: string    // 註冊日期（ISO 格式）
}
```

### 3. 留言資料（儲存在文章內）
```javascript
{
  content: string,     // 留言內容
  date: string,        // 留言時間
  userId: number,      // 留言者ID
  userEmail: string    // 留言者郵箱
}
```

## 部署說明
1. 專案使用純前端技術開發，無需後端伺服器
2. 所有外部依賴都使用 CDN 版本
3. 可直接部署到 GitHub Pages：
   - 建立 GitHub repository
   - 上傳所有檔案到主分支
   - 在 repository 設定中啟用 GitHub Pages

## 開發注意事項
1. 保持使用 CDN 版本的外部套件，確保能夠直接部署到 GitHub Pages
2. 新增功能時需考慮 localStorage 的儲存限制：
   - 圖片建議進行壓縮後再儲存
   - 定期清理過期的數據
3. 維持響應式設計，確保在各種裝置上都能正常顯示
4. 確保資料安全：
   - 所有使用者輸入都需要進行 XSS 防護
   - 密碼需要進行加密處理
5. 顏色主題：
   - 主要使用馬卡龍色系
   - 確保文字與背景的對比度符合無障礙標準
